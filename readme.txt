2014年5月28日
添加了如下功能：
1、文件进行分块发送，丢了块也可以通过发送块反馈包收到。
2、文件可按块为单位写，当收到一个块以后，就写在块列表中，当一个文件的所有块都收完全了，就从列表中一块块的读出来，然后写到文件中。
3、先发数据包，收到反馈包后再发冗余包。
4、解决了本地内存溢出问题。encode,decode,file_blocks函数都上了锁。
5、将手机屏幕固定，不再可以转屏幕

2014年6月4日
程序进行了改进：
1、一块发完再发下一块，一个文件发完再发下一个（发送完2s内没有收到反馈包，则认为发送成功）
2、多个小文件合并了再发送，建立了发送队列，实时监控队列
3、收到反馈包以后，对块编码，然后保存下来，直到收到对新的文件块的丢失反馈包，才删除
4、收到大于10m的文件，则直接将文件块写出。

2014年6月17日
1、文件队列触发的问题：最大容量：10m, 最小间隔：3s, 文件个数：20个
2、添加了发送目录的功能。
3、<100k的文件凑齐了发送的功能已经调通，并分别对 497B，36k,80K的文件做了实验。
4、测试了发送和收到文件的MD5值，是相同的。

2014年8月14日
1、为了和Seafile的android端程序配合，将共享的文件夹改为Seafile。
2、两个程序结合的适用，实现了普通节点上传文件到服务器的功能。
3、服务器上的文件也可以通过特殊节点，传到普通节点上。
4、可以多节点共享文件，新加入的节点可以主动要求与特殊节点同步。
5、问题：特殊节点不能与服务器同步（共享），只能在刷新时从服务端下载文件，还有上传到服务器时有限制条件。
